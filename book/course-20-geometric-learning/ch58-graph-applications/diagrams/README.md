# Chapter 58 Diagrams: Applications of Graph ML

This directory contains all educational diagrams for Chapter 58, created using matplotlib with a consistent color palette and style.

## Diagrams

### 1. domain_overview.png

- Clean, educational layout with consistent styling

### 2. molecular_workflow.png
**Location in content.md:** Line 109 (Examples section)
**Purpose:** Illustrate the three-step workflow for molecular property prediction
**Type:** Process diagram (matplotlib)
**Key Features:**
- Panel 1: Molecule represented as graph (atoms=nodes, bonds=edges)
- Panel 2: Message passing aggregation from neighbors
- Panel 3: Global pooling to graph embedding and final prediction
- Visual demonstration of GNN forward pass for molecules

### 3. fraud_detection_graph.png

**Key Features:**

- Shared device as key signal for fraud detection
- Legend explaining node types and colors

### 4. knowledge_graph_embeddings.png

### 5. recommendation_bipartite.png

## Color Palette

All diagrams use a consistent color scheme:
- **Blue (#2196F3)**: Primary color, users, general nodes
- **Green (#4CAF50)**: Positive/correct, items, success
- **Orange (#FF9800)**: Relations, warnings, highlights
- **Red (#F44336)**: Fraud/negative, errors, problems
- **Purple (#9C27B0)**: Methods, special features
- **Gray (#607D8B)**: Neutral, baselines, secondary elements

## Style Guidelines

- **Font size:** Minimum 12pt for readability
- **Resolution:** 150 DPI
- **Max width:** 800px
- **Background:** White
- **Line width:** 1.5-2.5px for clarity
- **Alpha transparency:** 0.3-0.7 for overlays
- **Grid:** Enabled with 0.3 alpha for data plots

## Generation

All diagrams are generated by `generate_diagrams.py`, which can be run to regenerate all figures:

```bash
python generate_diagrams.py
```

This ensures consistency and reproducibility across all chapter diagrams.

## Integration

Diagrams are integrated into `content.md` using relative paths:
- Format: `![Description](diagrams/filename.png)`
- Each diagram has a figure caption explaining its purpose
- Diagrams are numbered sequentially (Figure 1-8)
- Placed at logical points in the content flow
